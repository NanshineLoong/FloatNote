# ADR 0003：以 `_inbox.md` 识别项目空间

状态：接受。

一个工作目录内既有项目化笔记，也可能有历史遗留的松散 Markdown 文件。需要一个无需额外数据库或元数据文件、且用户可直接理解的格式边界。

决定：含有 `_inbox.md` 的子目录即为项目空间。`_tasks.md` 是项目任务清单，按首次需要懒创建；任何不以下划线开头的 `.md` 文件都是 piece，首个 piece 同样按需创建。以下划线前缀保留给系统文件。工作目录根部的松散 `.md` 保持兼容，作为独立文档而非项目空间。

后果：项目发现只检查 `_inbox.md`；用户可以直接用文件系统查看和迁移内容；新增系统文件必须使用 `_` 前缀，以免与 piece 混淆。
