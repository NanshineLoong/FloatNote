# README 截图素材

这个目录用于存放根目录 `README.md` 的产品截图。截图准备完成后，按下列文件名放入本目录，并将根 README 中对应的注释替换为图片引用。

| 文件名 | 展示内容 |
| --- | --- |
| `01-hero.png` | FloatNote 悬浮在真实工作场景中的完整产品印象图 |
| `02-capture.png` | 同一画面中展示悬浮笔记窗与划词后的采集、翻译、提问操作 |
| `03-dual-pane.png` | 采集区与写作区并排的双栏工作区 |
| `04-socratic-ai.png` | AI 追问过程，以及修改内容前的变更确认 |
| `05-local-files.png` | FloatNote 项目空间与本地 Markdown 文件 |
| `06-tags.png` | 为采集内容添加和查看标签 |
| `07-action-menu.png` | 行动菜单与下一步行动 |
| `08-versions.png` | 文章版本列表或版本预览 |

## 截图规范

- 使用同一套虚构演示内容，保持窗口尺寸、缩放比例和视觉主题一致。
- 宽图建议至少 1600px 宽；标签、行动菜单和版本管理三张局部图保持相同纵横比。
- 裁掉与功能无关的桌面区域，但为悬浮窗和菜单保留足够的使用场景。
- 不要出现真实姓名、私人笔记、文件路径、API Key、访问令牌或其他敏感信息。
- 优先使用 PNG；如果图片体积明显影响 README 加载，再统一转换为 WebP。
