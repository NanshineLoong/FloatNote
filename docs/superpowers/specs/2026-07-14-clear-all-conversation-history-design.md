# 全量清理对话记录设计

## 目标

在“查看全部对话”窗口右上角扫帚菜单中提供“清理全部对话记录”，让用户可以删除所有本地保存的对话与对应会话文件。

## 交互

- 菜单在“清理 7 天前的记录”“清理 30 天前的记录”后新增红色危险项“清理全部对话记录”。
- 点击后显示一次原生警告确认：“将删除所有对话。删除后无法恢复。”；取消时不修改数据。
- 成功后列表重新加载；若当前活动会话被删除，向主窗口发送 `chat://deleted`，以清理活动会话状态；随后发送 `chat://history-changed` 更新其他视图及托盘菜单。

## 实现

复用既有 `chat_clear_before_entries(timestamp)` Tauri 命令，传入 `Number.MAX_SAFE_INTEGER`。它会返回被删除的会话条目，前端据此判断活动会话是否受影响。该时间戳远大于 JavaScript `Date.now()` 的毫秒时间戳，因而覆盖所有正常保存的对话。

不新增后端命令、DTO 或持久化格式；现有 Rust 清理逻辑已经删除 session 文件、保存索引并刷新托盘菜单。

## 测试

为菜单项工厂提取可测试的纯 UI 辅助函数。JSDOM 回归测试断言全量清理项有危险样式并在确认时调用清理回调；Rust 现有清理逻辑测试继续覆盖存储删除行为。完整 `npm test` 与 `npm run build` 验证前端和侧车。
