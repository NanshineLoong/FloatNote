# FloatNote 设置窗口与设计系统升级设计

**日期：** 2026-07-14  
**状态：** 已确认  
**范围：** 先完整重做设置窗口，并把可复用的视觉规则沉淀到现有设计系统；主笔记、历史和划词弹窗不在本轮重排。

## 目标

本轮把设置窗口做成 FloatNote 后续 UI 迁移的设计样板，同时修正窗口外壳、Skills 管理和划词悬浮窗设置的行为问题。

交付结果必须满足：

- 设置窗口在 macOS 使用原生红绿灯，头部空白区域可拖动；窗口可缩放、最大化，并有明确最小尺寸。
- 视觉采用“FloatNote 融合版”：借鉴 CodePilot 的克制卡片、留白和信息层级，保留 FloatNote 的靛蓝语义色。
- “通用”页只保留“开机启动”。主题、界面字号、后台运行、关闭行为等项目都不出现。
- “AI 导师”更名为“AI”。
- Skills 始终展示内置项；导入操作选择一个根目录，且根目录必须包含 `SKILL.md`。
- “划词弹窗”快捷键并入“划词悬浮窗”；只有触发方式选择“快捷键”后才显示快捷键录制器。
- 下拉框、卡片、开关、快捷键录制器和错误状态形成一致、可访问的组件语言。

## 非目标

- 不在本轮重做主笔记窗口、历史窗口或划词弹窗的整体布局。
- 不建设 Skill 编辑器、Skill 市场、导入项删除功能或在线下载能力。
- 不新增“后台运行”“关闭行为”“启动后隐藏”等通用选项。
- 不引入 React、Tailwind、Radix 或新的 UI 框架；继续使用 Vanilla TypeScript、原生语义控件和项目现有 CSS token。
- 不照搬 CodePilot 的产品结构、主题系统或 Electron 平台实现。

## 选定方向与备选方向

选择 **B：FloatNote 融合版**。

- 采用 CodePilot 的柔和描边卡片、内缩分隔线、紧凑侧栏和有节奏的留白。
- 保留 FloatNote 现有靛蓝色作为选中、开关、焦点和主操作的唯一强调色。
- 内容层保持不透明；半透明或材质效果只允许用于平台外壳和侧栏。

未选择的方向：

- **CodePilot 忠实版：** 黑白对比更强，但会削弱 FloatNote 的产品识别。
- **macOS 浮层版：** 独立悬浮卡片更有材质感，但空间利用率较低，也不利于 Windows 一致性。

## 信息架构

设置窗口继续保留三个一级分类：

1. **通用**
   - 仅显示“开机启动”。
2. **AI**
   - AI 提供商。
   - Skills。
3. **快捷键**
   - 全局快捷键。
   - 划词悬浮窗触发方式。
   - 窗口快捷键。

侧栏文案必须使用“AI”，不再出现“AI 导师”。分类切换不重建窗口，保留当前设置窗口内的滚动和焦点语义。

## 窗口外壳

### 尺寸与缩放

- 默认尺寸：`780 × 620`。
- 最小尺寸：`720 × 520`。
- 设置窗口必须允许用户正常缩放和最大化。
- 内容在最小尺寸下不能产生水平滚动；右侧内容区采用受控最大宽度，剩余空间作为安全留白。

### macOS

- 使用 Tauri 原生窗口装饰与 Overlay 标题栏，显示系统红绿灯。
- 顶部提供约 38px 高的拖动区域，并为红绿灯保留安全间距。
- 只有纯外壳区域带拖动属性；按钮、侧栏、下拉框、开关、快捷键录制器和输入框必须明确为非拖动区域。

### Windows

- 使用 Windows 原生最小化、最大化和关闭控件，不模拟 macOS 红绿灯。
- 维持与 macOS 相同的信息架构、卡片组件和内容间距，仅平台外壳不同。

## 视觉系统

### 表面层级

- **窗口背景：** 中性底色。
- **标题栏与侧栏：** 可使用平台外壳表面 token；macOS 可轻微半透明，Windows 使用稳定实色回退。
- **内容画布和设置卡片：** 始终不透明，避免玻璃效果影响阅读。
- **卡片：** 中等圆角、柔和边框、无默认阴影；单行或多行设置统一使用内缩分隔线。
- **嵌套区域：** 使用比卡片更弱的中性色表面和更小圆角，不能叠加重阴影。

### 色彩与状态

- 靛蓝只承担 active、checked、focus 和 primary action。
- 普通边框、说明文字和禁用状态使用语义中性色。
- 错误信息使用 danger token，并配合文字或图标，不能只靠颜色表达。
- 明暗外观只跟随操作系统；设置窗口不提供应用内主题选择。

### 控件

- 原生 `<select>` 保留语义和键盘行为，但统一外观：稳定高度、规范内边距、单一箭头、明确 hover/focus/disabled 状态。
- 开关扩大到可用点击区域，原生 checkbox 继续承载可访问状态。
- 快捷键录制器呈现 idle、recording、saved、error 和 disabled 状态。
- 所有交互控件保留全局 `focus-visible` 环，动画使用现有 motion token，并尊重 `prefers-reduced-motion`。

本轮新增或调整的通用规则进入现有三层 token 架构：Primitive → Semantic → Component。设置页业务 CSS 不直接消费 primitive 色值，也不新建第二套主题系统。

## 通用设置

“通用”页面只保留一个“启动”分区和“开机启动”设置项。

- 不显示主题选择。
- 不显示界面字号步进器。
- 不显示后台运行、关闭行为或其他推测性设置。
- 页面即使内容较少，也保持与其他分类一致的标题、段落宽度和卡片节奏，不使用无意义装饰填充空间。

### 配置清理

- 从 Rust `Config`、前端 DTO、设置页事件绑定和 appearance 事件中删除 `theme` 与 `font_size`。
- 删除应用级主题偏好和可配置界面字号逻辑；所有窗口使用系统明暗偏好与设计系统默认字号。
- 删除界面字号的增减入口及其持久化行为，包括原设置页展示的 `Cmd/Ctrl +`、`Cmd/Ctrl -` 调整链路。
- 读取旧配置时，Serde 忽略遗留的 `theme`、`font_size` 键；下一次保存不再写回这些键，不需要破坏性迁移文件。
- 外观初始化只负责系统主题所需的静态样式，不再读取配置或监听 `appearance-changed`。

## AI 与 Skills

### AI 提供商

现有固定提供商配置和启用逻辑保持不变，只迁移到新的设置卡片与控件视觉。保存失败必须在对应提供商卡片内显示原因。

### Skill 目录来源

Skills 目录由 Tauri 主进程直接发现并生成目录清单，不依赖 sidecar 在线状态：

1. 打包资源目录中的内置 Skills。
2. 开发环境下 `src-tauri/resources/skills` 回退目录。
3. `~/.floatnote/skills` 下的用户导入 Skills。

当前内置项必须始终可见：

- `socratic-review`
- `structure-piece`
- `inbox-to-actions`

列表 DTO 至少包含：`name`、`description`、`source` 和 `enabled`。`source` 取 `builtin` 或 `imported`，前端显示“内置”或“已导入”标签。

内置与导入 Skill 都可以启用或停用；内置 Skill 不提供删除操作。本轮也不为导入 Skill 增加删除操作。

### 列表与运行时解耦

- Skills 列表命令直接读取目录并返回结果，因此 sidecar 未连接、启动失败或重启中都不影响列表展示。
- 启用状态仍由 `disabled_skills` 持久化。
- 用户修改启用状态后先保存配置，再请求 sidecar 重载。
- 若 sidecar 离线，保存仍成功，卡片显示“运行时恢复后生效”的非阻塞提示；运行时恢复时以最新配置重新加载。

### 导入流程

- 文件选择器只能选择一个目录，不能选择单个 Markdown 文件。
- 所选目录根部必须存在名为 `SKILL.md` 的普通文件，大小写必须准确。
- `SKILL.md` 必须包含非空 `name` 与 `description`；`name` 只允许 ASCII 字母、数字、`-` 和 `_`。
- Skill 名称不能与任何内置或已导入 Skill 冲突。
- 导入复制整个目录，以保留 `SKILL.md` 引用的脚本、参考资料和素材。
- 复制过程拒绝符号链接，避免导入越过用户明确选择的目录边界。
- 导入先复制到用户 Skills 根目录中的临时目录，完整成功后再原子重命名为最终目录，避免失败后留下半成品。
- 成功后立即刷新主进程目录清单，并请求 sidecar 重载。

导入错误必须说明具体恢复方式，例如“所选目录根部缺少 `SKILL.md`”“Skill 缺少 description”或“同名 Skill 已存在”。

## 快捷键与划词悬浮窗

### 全局快捷键

继续保留：

- 划线引用。
- 显示 / 隐藏。
- 划词悬浮窗使用的快捷键值（改为条件显示）。

### 合并触发方式

删除独立常驻的“划词弹窗”快捷键行，改为单个“划词悬浮窗”设置卡。

触发方式下拉选项为：

- 自动弹出
- 快捷键
- 关闭

只有值为 `shortcut` 时，卡片下方才显示“唤起划词弹窗”快捷键录制器。切换到 `auto` 或 `off` 时收起录制器，但保留已有 `shortcut_popup` 值，用户切回“快捷键”后可继续使用。

“自动弹出”的选项标签不附加“仅 macOS”。当前自动捕获仍由 macOS selection monitor 实现；不支持的平台在说明文字或禁用反馈中明确当前能力边界，不能静默表现为已经生效。

展开和收起只使用 transform/opacity 或离散显示状态，不能通过长时间高度动画阻塞交互，并在 reduced-motion 下关闭过渡。

### 冲突与保存

- 快捷键冲突在所有相关字段旁显示，而不是只标记窗口快捷键一侧。
- 存在冲突时不调用运行时应用命令，也不持久化冲突值。
- 应用失败时保留用户当前录入内容，显示可操作错误，不整页重绘丢失焦点。

## 前端模块边界

当前集中的设置页入口应拆成职责清晰的模块：

- **入口与外壳：** 获取配置、装配标题栏/侧栏/内容区、协调分类切换和顶层错误边界。
- **通用：** 只负责开机启动。
- **AI 提供商：** 继续复用现有 provider profile 与保存接口。
- **Skills：** 渲染目录、启停、导入和局部状态反馈。
- **快捷键：** 管理录制器、触发方式的渐进披露、冲突校验与应用。

各模块通过显式回调或小型接口调用保存逻辑，不直接互相查询 DOM。窗口拖动区和非拖动区由外壳模块统一声明。

## 错误与状态反馈

- 开机启动失败：显示在“开机启动”卡片下方，并回滚开关视觉状态。
- 提供商保存失败：显示在对应提供商卡片内。
- Skill 清单读取失败：Skills 分区显示重试操作；不能误显示为空清单。
- Skill 导入失败：显示具体目录或元数据问题，列表保持原状。
- sidecar 离线：Skills 清单仍显示；启停或导入保存后显示“运行时恢复后生效”。
- 快捷键冲突：每个冲突字段附近显示同一冲突关系。
- 页面级错误区域只处理无法载入基础配置等阻断性错误，不承接所有局部错误。

## 测试与验证

### TypeScript

- 通用页只包含“开机启动”，不含主题、界面字号、后台运行或关闭行为。
- 侧边栏使用“AI”。
- 划词悬浮窗为 `shortcut` 时显示录制器，`auto` 与 `off` 时隐藏，并保留快捷键值。
- 所有冲突字段都得到错误反馈，冲突状态不调用应用和保存。
- Skills 正确渲染来源、描述、启用状态、离线提示和局部错误。
- 设置页关键控件具备 label、role/state 与键盘可达性。

### Rust

- 内置目录扫描在打包路径和 debug 回退路径下工作。
- sidecar 未连接时仍能列出内置 Skills。
- 用户导入 Skills 与内置项合并，并具有稳定来源标记和顺序。
- 缺失或命名错误的 `SKILL.md`、缺失元数据、非法名称、符号链接和重名导入均被拒绝。
- 导入失败不留下最终目录或半成品。
- 旧配置含 `theme`、`font_size` 时可以读取，新保存结果不含这两个键。

### 窗口与视觉

- Tauri 配置或窗口测试断言设置窗口默认尺寸、最小尺寸、可缩放和平台标题栏策略。
- 在 `720 × 520`、默认尺寸和更大尺寸下检查无水平滚动、内容不遮挡。
- 检查系统浅色与深色、键盘导航、focus-visible、disabled 和 reduced-motion。
- macOS 实机检查红绿灯、拖动、最小化、最大化和缩放。
- Windows 实机或 CI 构建检查原生窗口控件与最小尺寸；无法在本机完成时，在交付说明中明确未执行项。
