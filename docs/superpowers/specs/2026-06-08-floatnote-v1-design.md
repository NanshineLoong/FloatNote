# FloatNote v1 设计文档

- 日期：2026-06-08
- 状态：已确认，待实现

## 1. 产品定位

FloatNote v1 是一个**始终置顶的悬浮 Markdown 速记窗口**，风格贴近 macOS 备忘录。核心价值：在阅读（或任意应用）时，用全局快捷键把划线原文以引用格式低摩擦地收进当前笔记，不打断心流；笔记以 Markdown 文件存到用户指定的目录和文件名。

v1 刻意**不做**：浏览器页面上下文绑定（URL/标题）、语音转文字、收件箱/瞬间分类、知识图谱、多 Agent。这些来自 Garden 产品文档的后续演进方向，不属于本版本。

成功标准：阅读中捕捉的注意力成本显著低于「切到笔记软件粘贴」。

## 2. 技术栈

- **Tauri v2**：单窗口桌面应用。
- **Rust 后端**：全局快捷键、窗口置顶/显隐、划线文字抓取、Markdown 文件读写、配置持久化。
- **Web 前端**：Markdown 编辑器（CodeMirror 6）+ 备忘录风格 UI + 设置面板。
- 前后端通过 Tauri command（前端调后端）和 event（后端通知前端，如「已追加引用」）通信。

主要依赖：`tauri-plugin-global-shortcut`、剪贴板能力（Tauri clipboard）、文件读写（Rust std 或 `tauri-plugin-fs`）。前端：CodeMirror 6（markdown 语言包 + 自定义主题）。

平台：v1 仅针对 macOS（划线抓取、辅助功能权限、窗口置顶均按 macOS 实现）。

## 3. 核心交互：两个全局快捷键

| 功能 | 默认键（可在设置改） | 行为 |
|---|---|---|
| 划线引用 | `⌥⌘C` | 抓取当前划线文字 → 格式化为引用块 → **静默追加**到当前笔记末尾 → **不抢焦点**。窗口隐藏时也照常写入。 |
| 显示/隐藏窗口 | `⌥⌘N` | 切换 FloatNote 窗口显隐。窗口可见时常驻置顶。 |

### 划线抓取机制（决策：模拟 Cmd+C + 读剪贴板）

按下「划线引用」时：

1. 备份当前系统剪贴板内容。
2. 模拟发送 `Cmd+C` 给前台应用。
3. 短暂等待（如 ~80ms）后读取剪贴板。
4. 若剪贴板**有新文字** → 进入引用格式化与追加流程。
5. 若剪贴板**无变化 / 为空**（说明没有选中）→ 静默忽略，不写入笔记。
6. 还原步骤 1 备份的剪贴板。

需要 macOS 辅助功能（Accessibility）权限来模拟按键。首次触发若未授权，引导用户去「系统设置 → 隐私与安全性 → 辅助功能」开启。

### 引用格式（决策：纯引用块）

- 单行原文：`> 原文`
- 多行原文：每行加 `> ` 前缀。
- 追加时在前后留一个空行，与已有内容隔开。
- 不附带来源应用名、不附带时间戳（保持最干净）。

## 4. 笔记模型

- 任意时刻存在唯一「**当前笔记**」，对应一个 `.md` 文件。
- 编辑器内容**实时自动保存**到该文件，采用防抖写盘（~500ms 空闲后落盘）。
- 每个笔记记住自己的**目标目录 + 文件名**。
- **文件名由笔记标题派生**：标题 `读书笔记` → 文件 `读书笔记.md`。标题改名时对应重命名文件（旧文件删除/移动到新名）。
- 存在一个可配置的**默认根目录**：首次启动让用户选择（建议默认 `~/FloatNote/`）。新建笔记默认落在根目录，也可为单个笔记单独指定其他目录。
- 窗口内提供**新建 / 切换笔记**入口：一个轻量下拉或侧栏，列出根目录下的 `.md` 文件；选中即成为当前笔记。

### 设置项

- 默认根目录路径。
- 两个全局快捷键的绑定。
- （可选）字体大小。

配置持久化到 Tauri app config 目录下的一个 JSON 文件。

## 5. UI / 视觉

- 单窗口，小尺寸（如默认 380×520，可调整、记住上次尺寸/位置）。
- 备忘录风格：干净留白、舒适字体与行距、轻量 Markdown 语法高亮（标题、引用块、列表、加粗等淡淡区分），浅色为主，跟随系统深浅色。
- 顶部极简栏：当前笔记标题（可点改名）+ 笔记切换/新建 + 设置入口。
- 主体：CodeMirror 编辑区。
- 无多余 chrome，贴合「极小悬浮」气质。

## 6. 错误处理

- **抓不到划线文字**：静默忽略，还原剪贴板，不污染笔记（不写空引用块）。
- **辅助功能权限缺失**：首次触发时检测并引导去系统设置开启；未授权时划线引用功能降级（提示）。
- **文件写入失败**（目录不存在/无权限/磁盘满）：前端明确提示错误，编辑器内未保存内容**绝不丢弃**，可重试。
- **标题重命名冲突**（目标文件名已存在）：提示用户，不覆盖已有文件。
- **快捷键注册冲突**（被其他应用占用）：设置界面提示注册失败，允许改键。

## 7. 测试策略

- **Rust 单元测试**：
  - 引用块格式化（单行 / 多行 / 含空行 / 含已有 `>` 的原文）。
  - 文件名从标题派生（含特殊字符、空格、中文的清洗规则）。
  - 配置读写（默认值、缺字段兼容）。
- **前端逻辑测试**：
  - 自动保存防抖触发。
  - 笔记切换时正确加载/落盘。
- **手动验证清单**（系统集成，写入 spec 验证步骤）：
  - 在 Safari / Preview / VS Code 划线后按快捷键，引用正确进入当前笔记且剪贴板被还原。
  - 窗口隐藏时划线引用仍写入。
  - 置顶在多桌面/全屏应用前的表现。
  - 辅助功能权限未授予时的引导流程。

## 8. 项目结构

```
FloatNote/
├── src-tauri/          # Rust 后端
│   ├── src/
│   │   ├── main.rs
│   │   ├── shortcuts.rs    # 全局快捷键注册与分发
│   │   ├── capture.rs      # 模拟 Cmd+C + 剪贴板抓取 + 还原
│   │   ├── quote.rs        # 引用块格式化（纯逻辑，可测）
│   │   ├── notes.rs        # 笔记文件读写、文件名派生、列表
│   │   └── config.rs       # 配置读写
│   └── tauri.conf.json
├── src/                # Web 前端
│   ├── editor/         # CodeMirror 集成 + 备忘录主题
│   ├── notes/          # 当前笔记状态、切换、自动保存
│   └── settings/       # 设置面板
└── docs/superpowers/specs/
```

## 9. 明确不在 v1 范围内

浏览器 URL/标题绑定、语音转文字、阅读瞬间收件箱与分类、跨平台（Win/Linux）、多笔记同屏、富文本 WYSIWYG、知识图谱/AI 整理。
