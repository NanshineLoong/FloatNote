# 多级列表折叠交互设计

## 目标

调整正文编辑器的多级 Markdown 列表折叠控件，使其接近参考样式，同时让折叠操作不改变编辑器光标或选区。

## 范围

仅调整非大纲模式的 `src/note/list-fold.ts` 与对应 CSS。大纲模式继续使用自己的折叠控件；列表 Markdown 文本和已有的折叠状态映射不改变。

## 交互

- 仅含嵌套列表的父项提供折叠入口；叶子项保持普通编辑行为。
- 父项有两个等价入口：列表标记左侧的箭头，以及原生 Markdown 标记位置。无序项对应圆点位置，有序项对应“数字 + 点”位置。
- 两个入口都调用同一个折叠切换逻辑。折叠状态由 `aria-expanded` 公开；键盘用户可通过箭头按钮的 Enter 或 Space 操作。
- 在 `mousedown` 阶段拦截入口事件，并让 widget 忽略 CodeMirror 的默认事件处理，防止点击折叠时移动光标、修改选区或夺取编辑器焦点。

## 视觉

- 箭头使用右/下方向的简洁 chevron；命中区比图标本体大，且更接近列表标记，避免正文布局跳动。
- 不使用硬编码紫色。静止状态采用 `--color-text-subtle`，悬浮状态采用 `--color-text`；键盘焦点环采用 `--color-accent`，与现有设计系统及浅/深色主题一致。
- 悬浮在箭头或列表标记入口时显示圆形描边/柔和背景反馈；父项标记可显示为实心或强调色，以清晰表达其可折叠性。叶子项不显示该反馈。

## 实现边界

保留原生 Markdown 标记的可编辑和渲染职责，通过 widget/装饰增加命中层；不将 `-`、`1.` 替换为自定义文本 widget。这样不会破坏 CodeMirror 的光标定位、复制、Markdown 语义或有序列表编号。

## 验证

- 为控件 DOM 结构与折叠状态补充单元测试。
- 覆盖箭头和标记入口均委托至同一个切换逻辑，以及 `mousedown` 被拦截的无选区副作用约束。
- 运行受影响 Vitest 文件及完整前端测试；手动验证无序和有序父项在浅/深色主题中的 hover、focus 与光标保持行为。
