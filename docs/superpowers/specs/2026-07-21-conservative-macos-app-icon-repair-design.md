# FloatNote macOS 应用图标保守修复设计

## 目标

采用已确认的“B · 保守修复”方向：消除 Launchpad、Finder 和 Dock 中的白色外围轮廓，同时最大限度保留 FloatNote 原始图标的纸张、花括号、笔触纹理、配色和立体阴影。方案 C 的矢量重绘不再使用。

## 原始素材

- 从提交 `f4454a9` 中恢复原始 `icon.icns`，使用其中的 1024×1024 档位作为修复基准。
- 不重新绘制纸张、花括号或书写线；这些前景像素以及原始阴影直接来自历史素材。
- Git 历史继续保存未经修复的原始图标，仓库中的新母版保存修复后的 1024×1024 成品。

## 修复方式

- 用原图蓝灰背景的顶部、侧边和底部颜色重建铺满 1024×1024 画布的不透明纵向渐变。
- 通过带羽化边缘的遮罩提取原始纸张、字符、线条及其投影，排除原图最外围的白色画布和预制圆角底板。
- 将提取的原始前景围绕画布中心放大 4%，保持原始旋转角度和视觉中心。
- 合成结果不预制 macOS 外层圆角；最终圆角继续由系统蒙版产生。
- 修复后的 `app-icon.png` 作为可重复生成 ICNS、ICO 和平台 PNG 的唯一母版。

## 资源与验证

- `npm run icon:generate` 从 `src-tauri/icons/app-icon.png` 生成现有平台资源。
- 删除方案 C 的 `app-icon.svg`，避免同时存在两套相互冲突的母版。
- 自动检查母版为 1024×1024、四角完全不透明且不接近白色、纸张中心仍为原始暖白色。
- 解包 ICNS，确认 16–1024px 的十个档位齐全；重新构建并挂载 DMG，确认应用包使用新 ICNS。

## 非目标

- 不重新设计品牌符号，不改变原始纸张材质、花括号形状和笔触风格。
- 不修改托盘图标、应用功能、Bundle ID、签名方式或 DMG 安装界面。
