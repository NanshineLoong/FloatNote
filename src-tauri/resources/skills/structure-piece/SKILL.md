# 组织成篇 piece

把 `_inbox` 里的散点聚类、排序，组织成一篇 piece 的结构化大纲（或草稿），可写入目标 piece 文件。

## 流程

1. 用 `read_note`（`target: { kind: "inbox" }`）读 `_inbox` 全文，识别散点。
2. 如已有目标 piece，用 `read_note`（`target: { kind: "piece", name: "<文件名>" }`，或缺省=当前活动 piece）读其现状。
3. 把散点**聚类**成主题，排成合理的**论证/叙述顺序**，给出大纲；必要时补成段落草稿。
4. 与用户确认大纲后，用 `edit_note` 把结构化内容写入目标 piece（`old_string` 选稳定锚点且唯一；插入或替换相应段落）。

## 你拥有的工具

- `read_note`：读目标笔记全文（target 缺省=当前活动笔记）。
- `edit_note`：唯一 str_replace，改块/插块/删块/改任务都用它。old_string 必须唯一。
- `write_note`：整篇覆写，仅大重构时用。

## 纪律

- 动手写**之前**，先给出大纲并说明**为什么这样组织**，让用户确认后再写。
- 所有写操作都会弹气泡让用户确认；用户可能拒绝，拒绝后不要反复重试。
- 不要擅自删除用户有意保留的原始想法；组织应服务于学习与表达，而非抹去原料。
- 只在确实有帮助时才写入；多数轮次应是讨论大纲，而非直接落笔。
